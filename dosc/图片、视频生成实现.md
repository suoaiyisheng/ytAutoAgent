要实现自动化，我们需要将原本“手动粘贴”的操作转变为**“数据驱动”**的流程。将参考图作为**变量**传入是保证 24 个分镜角色统一的关键。

以下是将原始流程转化为自动化算法的逻辑设计：

### 1. 数据结构化：定义“任务包”

首先，我们需要将你的 Markdown 表格和参考图转化成一个程序可读的任务列表（通常是 JSON 格式）。每一行分镜就是一个独立的对象。

```json
{
  "global_assets": {
    "ref_1": "path/to/purple_braid_woman.jpg",
    "ref_2": "path/to/muscular_man.jpg"
  },
  "tasks": [
    {
      "id": 1,
      "image_prompt": "1，真实摄影风格...参考图1女性...参考图2男性...",
      "video_prompt": "固定镜头...男性在水箱内挥动手臂...",
      "references": ["ref_1", "ref_2"]
    },
    {
      "id": 2,
      "image_prompt": "2，真实摄影风格...紫色长辫子女性...",
      "video_prompt": "固定镜头...紫色长辫子女性沿街道行走...",
      "references": ["ref_1"]
    }
  ]
}
```

---

### 2. 自动化执行逻辑（算法步骤）

#### 第一阶段：自动化生图（解决“手搓即梦”）

- **循环机制**：遍历 `tasks` 列表。
- **变量注入**：
- 读取 `image_prompt`。
- 检查 `references` 数组。如果包含 `ref_1`，则在请求参数中关联 `global_assets.ref_1` 的图片链接。

- **并发请求**：程序自动向生图接口发送 2 次请求（对应原流程的 2\*4=8 张图）。
- **自动存储**：将生成的 8 张图存入 `output/scene_1/` 文件夹。

#### 第二阶段：智能/手动筛选（解决“抽一张出来”）

这是目前自动化中最难的一步。

- **半自动方案**：程序弹出一个快速预览窗口，你点选最满意的一张，程序记录该图片 ID 为 `selected_image_1.png`。
- **全自动方案（进阶）**：使用人脸识别算法（如 Face Analysis）计算这 8 张图与 `ref_1` 的人脸相似度，自动选出得分最高的那张。

#### 第三阶段：自动化生视频（解决“海螺/Veo3 批量插件”）

- **输入绑定**：将上一步选出的 `selected_image_1.png` 作为“底图变量”。
- **提示词注入**：将 `video_prompt` 作为“动作变量”。
- **接口推送**：自动化脚本排队将 24 组“图片 + 提示词”推送到视频生成模型 API。

#### 第四阶段：自动化剪辑（解决“剪辑模板复用”）

- **协议替换**：不需要打开 CapCut 手动拖拽。
- **算法逻辑**：

1. 读取一个预先做好的“剪映草稿协议文件”（JSON 格式）。
2. 找到协议里的 `media_path`（素材路径）字段。
3. **变量替换**：将占位符路径替换为 `output/video_1.mp4`, `output/video_2.mp4` 等。
4. **自动渲染**：保存修改后的协议文件，利用命令行或工具直接导出视频。

### 3. 核心变量传递路线图

### 4. 总结：你需要的“自动化引擎”结构

| 变量名称        | 来源                    | 作用阶段                                               |
| --------------- | ----------------------- | ------------------------------------------------------ |
| `Character_REF` | `global_assets`         | **生图阶段**：锁定角色长相，解决“紫色长辫子”的一致性。 |
| `Scene_Prompt`  | `tasks[i].image_prompt` | **生图阶段**：描述环境、光影和构图。                   |
| `Generated_IMG` | 第一阶段输出            | **生视频阶段**：作为图生视频的起始帧。                 |
| `Action_Prompt` | `tasks[i].video_prompt` | **生视频阶段**：描述动态效果（如：水波、泪水、微笑）。 |

**原本你手动做这些需要反复切换窗口、复制粘贴。自动化后，你只需要运行脚本，然后进行一次“24 选 1”的确认即可。**
